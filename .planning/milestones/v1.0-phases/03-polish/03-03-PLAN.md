  <action>
    **设计原理：** Go build tags 是处理平台差异的标准机制（D-10）。将文件权限设置分离到平台特定文件中，比 `runtime.GOOS` 散弹枪检查更清晰。Windows NTFS 不支持 Unix 权限位，os.Chmod 在 Windows 上要么失败要么只影响只读标志，因此静默降级更安全（D-13）。权限设置在**下载完成**后应用到本地文件——因为 SFTP OpenRemoteFile 返回 io.ReadCloser 没有提供远程文件 mode 信息，而下载目标文件在本地可以通过 os.Stat 获取 mode。上传时的远程权限设置需要 `sftp.Chmod()` 额外接口调用，此处不做（超出 INTG-03 范围）。

    **实现步骤：**

    1. **创建 `internal/adapters/data/transfer/permissions_unix.go`**：
       ```go
       //go:build !windows

       package transfer

       import (
           "os"
           "go.uber.org/zap"
       )

       // setFilePermissions sets file permissions on Unix-like systems.
       // Logs a warning if chmod fails (e.g., permission denied).
       func setFilePermissions(path string, mode os.FileMode, log *zap.SugaredLogger) {
           if err := os.Chmod(path, mode); err != nil {
               log.Warnw("failed to set file permissions", "path", path, "mode", mode, "error", err)
           }
       }
       ```

    2. **创建 `internal/adapters/data/transfer/permissions_windows.go`**：
       ```go
       //go:build windows

       package transfer

       import (
           "os"
           "go.uber.org/zap"
       )

       // setFilePermissions is a no-op on Windows.
       // Windows NTFS does not support Unix-style permission bits.
       func setFilePermissions(path string, mode os.FileMode, log *zap.SugaredLogger) {
           log.Debugw("skipping chmod on Windows (NTFS)", "path", path)
       }
       ```
       两个版本函数签名一致（path string, mode os.FileMode, log *zap.SugaredLogger），Windows 版本忽略 mode 值。

    3. **修改 `transfer_service.go` 在下载完成后调用 setFilePermissions**：

       在 `downloadSingleFile` 或 `DownloadFile` 中，copyWithProgress 成功后设置本地文件权限。由于当前 DownloadFile 不获取远程文件 mode，采用以下方案：

       **方案：** 不从远程获取原始权限，而是在下载完成后通过 `os.Stat` 检查本地文件当前的 mode，确保它有合理的默认权限（0644 for files）。实际上这没有实际意义——下载的文件已经有默认权限。

       **修正方案（实用）：** setFilePermissions 主要作为跨平台兼容性基础设施。在下载完成后，对于从远程下载的文件，调用 `setFilePermissions(localPath, 0o644, ts.log)` 设置标准权限。这不是"保留原始权限"（那需要远程 Stat 获取 mode），而是确保下载文件有一个确定、合理的权限值。Unix 上会执行 chmod，Windows 上静默跳过。

       ```go
       // 在 DownloadFile 中 copyWithProgress 成功后
       if err == nil {
           setFilePermissions(localPath, 0o644, ts.log)
       }
       ```

       对于 `downloadSingleFile`（目录传输中的单个文件下载），同样添加。
