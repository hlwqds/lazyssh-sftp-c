Create file `internal/adapters/ui/file_browser/progress_bar.go` with package `file_browser`.

Imports: `"fmt"`, `"github.com/gdamore/tcell/v2"`.

Define `ProgressBar` struct:
```go
type ProgressBar struct {
    current  int64
    total    int64
    width    int
    fillChar rune // default: '█'
    emptyChar rune // default: '░'
    color    tcell.Color // default: tcell.Color248
}
```

Methods:
- `NewProgressBar() *ProgressBar` — constructor with defaults: fill='█', empty='░', color=Color248
- `(pb *ProgressBar) SetProgress(current, total int64)` — update progress values
- `(pb *ProgressBar) SetColor(color tcell.Color)` — change bar color
- `(pb *ProgressBar) SetWidth(width int)` — set explicit width
- `(pb *ProgressBar) String() string` — render progress bar as string

`String()` implementation:
- If total == 0: return empty string or "N/A" (no progress to show)
- Calculate percentage: `pct := float64(pb.current) / float64(pb.total)`
- If width == 0: use default width of 30
- Fill count: `fillCount := int(pct * float64(pb.width))`
- Build string: repeat fillChar fillCount times + repeat emptyChar (width-fillCount) times
- Use `strings.Repeat(string(rune), count)` for rendering

Helper: add package-level function `formatSpeed(bytesPerSec float64) string`:
- if bytesPerSec >= 1024*1024: return `fmt.Sprintf("%.1f MB/s", bytesPerSec/1024/1024)`
- else: return `fmt.Sprintf("%.1f KB/s", bytesPerSec/1024)`

Helper: add package-level function `formatETA(seconds float64) string`:
- if seconds <= 0 or > 86400: return ""
- mins := int(seconds) / 60
- secs := int(seconds) % 60
- return `fmt.Sprintf("%dm %ds", mins, secs)`

Create file `internal/adapters/ui/file_browser/transfer_modal.go` with package `file_browser`.

Imports: `"fmt"`, `"strings"`, `"time"`, `domain` package, `tcell`, `tview`.

Define `TransferModal` struct embedding `*tview.Box`:
```go
type TransferModal struct {
    *tview.Box
    app       *tview.Application
    bar       *ProgressBar
    // UI elements (tview.TextView)
    fileLabel *tview.TextView  // "Uploading: filename.txt"
    infoLine  *tview.TextView  // "67%  2.3 MB/s"
    etaLine   *tview.TextView  // "ETA: 0m 12s"
    summaryLine *tview.TextView // "Transferred 8/10 files, 2 failed" (for dir transfers)
    // Speed tracking
    speedSamples []speedSample
    // State
    visible   bool
    direction string // "upload" or "download"
    onDismiss func() // callback when modal is hidden
}

type speedSample struct {
    time  time.Time
    bytes int64
}
```

Constructor:
```go
func NewTransferModal(app *tview.Application) *TransferModal
```
- Create Box with padding (5,10,5,10 — top/right/bottom/left)
- Create ProgressBar with default width 30
- Create TextViews for fileLabel, infoLine, etaLine, summaryLine
- Set text colors: fileLabel Color255 (white), infoLine Color245, etaLine Color245
- Initialize speedSamples as empty slice with capacity 5
- Set border: Color238, title color Color250
- Set background: Color232
- Return TransferModal

`Draw(screen tcell.Screen)` method:
- Call `tm.Box.Draw(screen)` for background
- Get inner rect with `tm.GetInnerRect()`
- Calculate center positions for each text element
- Draw fileLabel centered at top
- Draw ProgressBar centered at middle (call `tm.bar.String()`)
- Draw infoLine (percentage + speed) to the right of progress bar or below it
- Draw etaLine below progress bar
- Draw summaryLine at bottom if visible (for dir transfer summary)

Actually, simpler approach using tview.TextView positioning via DrawText:
- Draw file name: centered, row = innerY + 1
- Draw progress bar + info: centered, row = innerY + 3
  - Left side: progress bar string
  - Right side: " 67%  2.3 MB/s"
- Draw ETA: centered, row = innerY + 5
  - "ETA: 0m 12s"

Use `tview.PrintSimple()` or manual screen cell setting for centered text.


`Show(direction, filename string)` method:
- Set `tm.visible = true`
- Set title: `" " + direction + " " + filename + " "` (direction = "Uploading" or "Downloading")
- Reset speed samples
- Reset bar progress
- Set fileLabel text
- Hide summaryLine
- Call `tm.app.SetRoot(tm, true)` to overlay

`Update(p domain.TransferProgress)` method:
- Update progress bar: `tm.bar.SetProgress(p.BytesDone, p.BytesTotal)`
- Calculate speed using sliding window:
  - Append `speedSample{time.Now(), p.BytesDone}` to samples
  - Keep only last 5 samples
  - If >= 2 samples: speed = (latest.bytes - oldest.bytes) / (latest.time - oldest.time).Seconds()
- Update infoLine: percentage + speed text
- Update etaLine: ETA from remaining bytes / speed
- If p.Done: set bar color to blue (Color33)
- If p.Failed: set infoLine to red error text
- If multi-file (p.FileTotal > 0): update title with "file 3/10"
- Request redraw: `tm.app.Draw()` (caller should use QueueUpdateDraw)

`ShowSummary(transferred, failed int, failedFiles []string)` method:
- Clear fileLabel, progress bar, infoLine, etaLine
- Set summaryLine text: `"Transferred %d/%d files" % (transferred, transferred+failed)`
- If failedFiles: append list of failed file names
- Set summaryLine color: green if failed==0, red otherwise
- Set title: " Transfer Complete "

`Hide()` method:
- Set `tm.visible = false`
- Call `tm.onDismiss()` if set (returns to file browser view)

`IsVisible() bool` — returns tm.visible

`SetDismissCallback(fn func())` — sets the onDismiss callback

`HandleKey(event *tcell.EventKey) *tcell.EventKey`:
- Esc: call `tm.Hide()`, return nil
- Any other key when in summary mode: call `tm.Hide()`, return nil
- Otherwise: return event (pass through)
