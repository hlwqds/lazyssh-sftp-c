## Interaction Contract

### Open Popup (POPUP-01)

1. User presses `r` in `handleGlobalKeys`
2. Guard: `fb.activePane == 1` AND `fb.remotePane.IsConnected()` AND no other overlay visible
3. Call `fb.recentDirs.Show()` -- sets `visible = true`, resets `selectedIndex = 0`
4. Return `nil` to consume event

### Navigate List (POPUP-02)

1. `j` / `KeyDown`: `selectedIndex++`, clamp to `len(paths)-1`
2. `k` / `KeyUp`: `selectedIndex--`, clamp to `0`
3. All other keys: return `nil` (consume, do nothing -- full key interception per D-08)

### Select Path (POPUP-03)

1. Enter pressed with non-empty list
2. `selectedPath := rd.GetPaths()[rd.selectedIndex]`
3. Execute sequence: `rd.Hide()` -> `remotePane.NavigateTo(selectedPath)` -> `rd.Record(selectedPath)` -> `app.SetFocus(remotePane)`
4. Source: CONTEXT.md D-10

### Close Popup (POPUP-04)

1. Esc pressed
2. Call `rd.Hide()` -- sets `visible = false`
3. No `app.Sync()` call (FileBrowser.Draw naturally redraws per D-09)
4. Return `nil` to consume event

### Empty State (POPUP-05)

1. `len(rd.paths) == 0`
2. Render "ćš‚ć— ćś€čż‘ç›®ĺ˝•" centered in popup at `tcell.Color240`
3. Minimum popup height = 5 cells to accommodate text
4. j/k/Enter are no-ops (list is empty)

### Current Path Highlight (AUX-01)

1. `currentPath := strings.TrimRight(remotePane.GetCurrentPath(), "/")`
2. For each path in list: if `path == currentPath`, use `tcell.ColorYellow` foreground
3. If current path is also the selected item, apply both `tcell.Color236` background AND `tcell.ColorYellow` foreground

### Rendering Order (CRITICAL -- Pitfall 3 prevention)

Per RESEARCH.md Pitfall 3, rendering order MUST be:

1. Call `rd.SetRect(x, y, width, height)` -- position the Box
2. Call `rd.Box.DrawForSubclass(screen, rd)` -- draw border, title, background fill
3. Get inner rect via `rd.GetInnerRect()`
4. For each list row:
   a. If selected: fill entire row with `tcell.Color236` background via `screen.SetContent`
   b. Then: `tview.Print(screen, path, x+1, row, width-2, AlignLeft, fgColor)` -- render text on top of background
5. For empty state: `tview.Print(screen, "ćš‚ć— ćś€čż‘ç›®ĺ˝•", x, centerY, width, AlignCenter, tcell.Color240)`

## Key Routing Architecture

### handleGlobalKeys Modification (RESEARCH.md Pattern 2)

```
handleGlobalKeys(event):
  1. IF recentDirs.IsVisible()  ->  return recentDirs.HandleKey(event)  // FIRST
  2. IF transferModal.IsVisible()  ->  transferModal.HandleKey(event); return nil  // existing
  3. switch KeyTab -> switchFocus
  4. switch KeyESC -> existing logic
  5. switch KeyF5 -> initiateDirTransfer
  6. switch Rune 's' -> cycleSortField
  7. switch Rune 'S' -> reverseSort
  8. switch Rune 'r' -> IF activePane==1 && remotePane.IsConnected() THEN recentDirs.Show(); return nil
  9. return event  // pass to pane
```

### HandleKey Method Signature

```go
func (rd *RecentDirs) HandleKey(event *tcell.EventKey) *tcell.EventKey
```

Returns `nil` for all handled keys (consumed). Never returns the event to caller -- full key interception per D-08.

Source: RESEARCH.md Pattern 1 + Pitfall 1 fix

func (fb *FileBrowser) Draw(screen tcell.Screen) {
    // 1. Fill background (existing)
    // ... existing ColorDefault fill loop ...
