**Step 1: Create `internal/core/ports/relay_transfer.go`**

Define the port interface:
```go
package ports

import (
    "context"
    "github.com/Adembc/lazyssh/internal/core/domain"
)

// RelayTransferService transfers files between two remote servers via local relay.
// Files are downloaded from the source SFTP to a local temp file/directory,
// then uploaded from the temp to the target SFTP.
type RelayTransferService interface {
    // RelayFile transfers a single file from source to target via local temp.
    // Download from srcSFTP -> temp file -> upload to dstSFTP.
    // Temp file is cleaned up on all code paths (success, error, cancel).
    RelayFile(ctx context.Context, srcPath, dstPath string,
        onProgress func(domain.TransferProgress),
        onConflict domain.ConflictHandler) error

    // RelayDir transfers a directory recursively from source to target via local temp.
    // Download from srcSFTP -> temp dir -> upload to dstSFTP.
    // Returns list of failed file paths (empty = all success).
    // Temp directory is cleaned up on all code paths.
    RelayDir(ctx context.Context, srcPath, dstPath string,
        onProgress func(domain.TransferProgress),
        onConflict domain.ConflictHandler) ([]string, error)
}
```

**Step 2: Create `internal/adapters/data/transfer/relay_transfer_service.go`**

Implement the adapter by composing two `transfer.New()` instances (one per SFTP connection). This follows the exact same pattern as CopyRemoteFile/CopyRemoteDir in transfer_service.go but uses two different SFTPService instances.

Key implementation details:
```go
package transfer

import (
    "context"
    "fmt"
    "os"

    "github.com/Adembc/lazyssh/internal/core/domain"
    "github.com/Adembc/lazyssh/internal/core/ports"
    "go.uber.org/zap"
)

type relayTransferService struct {
    log     *zap.SugaredLogger
    srcSFTP ports.SFTPService
    dstSFTP ports.SFTPService
}

func NewRelay(log *zap.SugaredLogger, srcSFTP, dstSFTP ports.SFTPService) *relayTransferService {
    return &relayTransferService{log: log, srcSFTP: srcSFTP, dstSFTP: dstSFTP}
}
```

RelayFile implementation:
1. `os.CreateTemp("", "lazyssh-relay-*")` -> close immediately, defer `os.Remove(tmpPath)`
2. Create `dlSvc := transfer.New(rs.log, rs.srcSFTP)` for download phase
3. Call `dlSvc.DownloadFile(ctx, srcPath, tmpPath, onProgress, nil)` -- nil onConflict for download (temp is always new)
4. Create `ulSvc := transfer.New(rs.log, rs.dstSFTP)` for upload phase
5. Call `ulSvc.UploadFile(ctx, tmpPath, dstPath, onProgress, onConflict)` -- pass onConflict for upload
6. Return nil on success, wrapped error on failure

RelayDir implementation:
1. `os.MkdirTemp("", "lazyssh-relaydir-*")` -> defer `os.RemoveAll(tmpDir)`
2. Create `dlSvc := transfer.New(rs.log, rs.srcSFTP)` for download phase
3. Call `dlSvc.DownloadDir(ctx, srcPath, tmpDir, onProgress, nil)` -- nil onConflict for download
4. Create `ulSvc := transfer.New(rs.log, rs.dstSFTP)` for upload phase
5. Call `ulSvc.UploadDir(ctx, tmpDir, dstPath, onProgress, onConflict)` -- pass onConflict for upload
6. Return `(failedFiles, nil)` on success, `(failed, err)` on failure

Do NOT add a compile-time interface check (unlike TransferService, this is not injected into existing DI).

Do NOT export the constructor via ports -- the UI layer (DualRemoteFileBrowser) will create it directly since it holds the two SFTP instances.
