## Spacing Scale

This is a terminal UI. Spacing is character-cell-based, not pixel-based. The following cell-level spacing conventions are established in the codebase:

| Token | Cells | Usage |
|-------|-------|-------|
| Compact | 0 | Inline prefixes (`[C] `, `* `) |
| Standard | 1 | Cell padding in Table columns |
| Section | 1 row | Separator rows in overlay dialogs |
| Status | 1 row | Status bar (fixed height at bottom) |
| Modal padding | 2,2,5,5 | TransferModal Box border padding (top, bottom, left, right) |

Exceptions: ConfirmDialog uses 60% terminal width (max 70 columns), 7-8 rows centered -- this is a responsive sizing pattern, not a fixed spacing value.

## Interaction Contract

### Key Bindings

| Key | Context | Action | Feedback |
|-----|---------|--------|----------|
| `c` | No overlay visible, pane focused | Mark current file as copy source | Status bar: "Clipboard: {filename}" in green; file row shows `[C]` prefix |
| `p` | No overlay visible, pane focused, clipboard active | Paste copied file to current directory | Local: instant (status bar flash). Remote: TransferModal shows copy progress |
| `p` | No overlay visible, clipboard empty | No action | Silent (no feedback) |
| `p` | No overlay visible, clipboard active, different pane | Reject cross-pane paste | Status bar error: "Cross-pane paste not supported (v1.3+)" |
| `Esc` | No overlay visible, clipboard active | Clear clipboard | Status bar: "Clipboard cleared" in green; `[C]` prefix removed from file |
| `Esc` | No overlay visible, clipboard empty | Close file browser | (existing behavior) |
| `c` | No overlay visible, clipboard already active | Replace old clipboard entry | Old clipboard cleared, new file marked (no confirmation needed) |

### Esc Priority Chain (updated)

1. InputDialog visible -> delegate to InputDialog.HandleKey
2. ConfirmDialog visible -> delegate to ConfirmDialog.HandleKey
3. RecentDirs visible -> delegate to RecentDirs.HandleKey
4. TransferModal visible -> delegate to TransferModal.HandleKey
5. Clipboard active -> clear clipboard, return nil (do NOT close browser)
6. Otherwise -> close file browser

### Clipboard State Machine

```
Empty --[c]--> Active(sourcePane, fileInfo, sourceDir)
Active --[c]--> Active(newSourcePane, newFileInfo, newSourceDir)  // replace
Active --[p, success]--> Empty  // auto-clear on success (D-05)
Active --[p, failure]--> Active  // keep for retry (D-05)
Active --[Esc]--> Empty  // explicit clear (D-05)
Active --[x]--> Active(OpMove)  // Phase 8: transition to move
```

### [C] Prefix Rendering Rules

Location: Name column (column 0) in LocalPane/RemotePane populateTable.

Rendering logic (checked AFTER existing `selected` map check):

1. If clipboard is active AND clipboard.FileInfo.Name matches current file AND clipboard.SourceDir matches current pane path:
   - Prepend `[C] ` to nameText (replacing any `* ` prefix if both states exist)
   - Set nameColor to `#00FF7F` (green)
2. Else if file is in `selected` map:
   - Prepend `* ` to nameText (existing behavior)
   - Set nameColor to `#FFD700` (gold)
3. Otherwise: normal rendering.

Priority: clipboard `[C]` takes precedence over Space `* ` selection. When both apply to the same file, show `[C]` only.

### Clipboard Access Pattern

The clipboard field lives on FileBrowser struct. LocalPane/RemotePane populateTable cannot directly access it (same package, so no import cycle, but no reference exists). Solution: add a `clipboardProvider func() (bool, domain.FileInfo, string)` callback to both pane types, injected during FileBrowser.build(). This follows the existing `onPathChange`/`onFileAction` callback pattern.

### TransferModal modeCopy

New mode value added to the existing `modalMode` enum:

```go
const (
    modeProgress       modalMode = iota
    modeCancelConfirm
    modeConflictDialog
    modeSummary
    modeCopy           // Phase 7: remote copy (download+re-upload) progress
)
```

**Draw behavior:** modeCopy reuses `drawProgress()` -- identical layout to modeProgress (file label + progress bar + speed + ETA). The difference is in the title and fileLabel text, set by a new `ShowCopy()` method.

**HandleKey behavior:** modeCopy shares the same HandleKey case as modeProgress (`case modeProgress, modeCopy:`). Esc triggers cancel confirmation, all other keys pass through.

**ShowCopy method:**
```go
func (tm *TransferModal) ShowCopy(filename string) {
    tm.visible = true
    tm.mode = modeCopy
    tm.cancelConfirmed = false
    tm.SetTitle(fmt.Sprintf(" Copying %s ", filename))
    tm.bar = NewProgressBar()
    tm.speedSamples = tm.speedSamples[:0]
    tm.fileLabel = fmt.Sprintf("Copying: %s", filename)
    tm.infoLine = ""
    tm.etaLine = ""
}
```

**Phase labels in progress:** During download phase, fileLabel shows `"Downloading: {filename}"`. During upload phase, fileLabel shows `"Uploading: {filename}"`. The onProgress callback in CopyRemoteFile sets these labels.

**Summary:** On completion, calls `ShowSummary(transferred, failed, failedFiles)` with title "Copy Complete" instead of "Transfer Complete". Reuse existing ShowSummary logic.

## File Operations Handler Pattern

### handleCopy (new)

Follows the established handler pattern (same as handleDelete/handleRename/handleMkdir):

1. Get active selection row and cell
2. Extract FileInfo reference
3. Validate: row >= 1, cell not nil, reference is FileInfo
4. For remote pane: check `IsConnected()`, show error if not
5. Create Clipboard struct: `Active=true, SourcePane=fb.activePane, FileInfo=fi, SourceDir=currentPath, Operation=OpCopy`
6. Call `fb.refreshPane(fb.activePane)` to trigger [C] prefix rendering
7. Call `fb.updateStatusBarTemp(...)` to show "Clipboard: {filename}" in green

### handlePaste (new)

Follows the goroutine + QueueUpdateDraw pattern:

1. Guard: `if !fb.clipboard.Active { return }` (silent, no feedback)
2. Guard: `if fb.clipboard.SourcePane != fb.activePane { showStatusError("Cross-pane paste not supported (v1.3+)"); return }`
3. Guard: for remote pane, check `IsConnected()`
4. Compute source path: `buildPath(clipboard.SourcePane, clipboard.SourceDir, clipboard.FileInfo.Name)`
5. Compute target path: `buildPath(activePane, currentPath, clipboard.FileInfo.Name)`
6. Same-directory check: if `currentPath == clipboard.SourceDir`, use `nextAvailableName()` to generate non-conflicting target
7. Dispatch:
   - **Local pane:** `go func() { err := fileService.Copy/CopyDir(src, dst); QueueUpdateDraw { if err { showStatusError; return }; clipboard = Clipboard{}; refreshPane; focusOnItem } }()`
   - **Remote pane:** Show TransferModal in modeCopy, then `go func() { err := transferSvc.CopyRemoteFile/CopyRemoteDir(ctx, src, dst, onProgress, onConflict); QueueUpdateDraw { handle success/failure } }()`
8. On success: clear clipboard (D-05), refresh pane, position cursor on copied file
9. On failure: show status error, do NOT clear clipboard (D-05 -- allow retry)
