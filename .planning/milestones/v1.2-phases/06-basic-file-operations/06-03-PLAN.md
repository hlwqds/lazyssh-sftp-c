    ## 2.1 handleDelete (DEL-01, DEL-02, DEL-03, DEL-04)

    逻辑：
    1. 根据 activePane 获取 paneIdx、fs (fileService 或 sftpService)、currentPath
    2. 远程面板需检查 IsConnected()
    3. 调用 pane.SelectedFiles() 获取多选文件列表
    4. 有多选 -> handleBatchDelete（DEL-03）
    5. 无多选 -> 获取当前选中项的 FileInfo，构建确认信息（D-03: 文件名、大小、类型、修改时间）
    6. 目录显示递归警告 detail（D-05: "Directory not empty, all contents will be deleted"）
    7. SetOnConfirm 回调：goroutine 中执行 Remove/RemoveAll，QueueUpdateDraw 中 refreshAndReposition
    8. 调用 confirmDialog.Show()

    ## 2.2 handleBatchDelete (DEL-03)

    逻辑：
    1. 计算总大小（跳过目录，per Research open question 3）
    2. 构建汇总消息："Delete N items? Total size: X.XM"
    3. SetOnConfirm 回调：goroutine 中逐个删除，记录首个错误
    4. QueueUpdateDraw 中 refreshPane + showStatusError（如有错误）

    ## 2.3 handleRename (REN-01, REN-02)

    逻辑：
    1. 远程面板检查 IsConnected()
    2. 获取当前选中项 FileInfo
    3. inputDialog.SetOnSubmit 回调：
       a. 空名称检查
       b. 无变化检查
       c. Stat 检查目标是否存在（REN-02）
       d. 目标存在 -> confirmDialog 二次确认（"Name Conflict"）
       e. 无冲突 -> goroutine 执行 Rename，QueueUpdateDraw refreshPane
    4. inputDialog.Show("Rename", "New name: ", fi.Name)

    ## 2.4 handleMkdir (MKD-01, MKD-02)

    逻辑：
    1. 远程面板检查 IsConnected()
    2. inputDialog.SetOnSubmit 回调：
       a. 空名称检查
       b. goroutine 执行 Mkdir，QueueUpdateDraw refreshPane + focusOnItem（MKD-02）
    3. inputDialog.Show("New Directory", "Directory name: ", "")

    ## 2.5 辅助方法

    添加以下方法：

    - `getActiveSelection() (int, int)` -- 根据 activePane 调用 localPane/remotePane.GetSelection()
    - `getActiveCell(row, col int) *tview.TableCell` -- 根据 activePane 调用 GetCell
    - `getCurrentPanePath() string` -- 根据 activePane 调用 GetCurrentPath
    - `getFileService() ports.FileService` -- activePane==0 返回 fileService，否则返回 sftpService
    - `refreshPane(paneIdx int)` -- 根据 paneIdx 调用 Refresh()
    - `refreshAndReposition(paneIdx int, deletedRow int)` -- 刷新后定位光标到 min(deletedRow, totalRows-1)，至少为 1（DEL-04）
    - `focusOnItem(paneIdx int, name string)` -- 遍历表格行查找匹配名称的 FileInfo，调用 Select 定位（MKD-02）

    ## 2.6 关键设计说明

    - 目录删除使用 RemoveAll（Pitfall 1：SFTP Remove 不能删除非空目录）
    - 所有文件操作在 goroutine 中执行（Pitfall 2：避免阻塞 UI）
    - joinPath 用于路径拼接（同 package 可用，已存在于 remote_pane.go）
    - 重命名冲突使用两步流程：InputDialog -> Stat 检查 -> ConfirmDialog（Pitfall 8：不会同时显示两个 overlay）
