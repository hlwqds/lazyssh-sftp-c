    Modify `handleGlobalKeys` to add overlay visibility checks and `r` key routing. The new structure (per RESEARCH Pattern 2 and UI-SPEC Key Routing Architecture):

    ```go
    func (fb *FileBrowser) handleGlobalKeys(event *tcell.EventKey) *tcell.EventKey {
        // Overlay key interception (check BEFORE any other key handling)
        if fb.recentDirs != nil && fb.recentDirs.IsVisible() {
            return fb.recentDirs.HandleKey(event)
        }
        switch event.Key() { //nolint:exhaustive
        case tcell.KeyTab:
            fb.switchFocus()
            return nil
        case tcell.KeyESC:
            if fb.transferModal != nil && fb.transferModal.IsVisible() {
                fb.transferModal.HandleKey(event)
                return nil
            }
            fb.close()
            return nil
        case tcell.KeyF5:
            fb.initiateDirTransfer()
            return nil
        }
        switch event.Rune() {
        case 'r':
            if fb.activePane == 1 && fb.remotePane.IsConnected() {
                fb.recentDirs.SetCurrentPath(fb.remotePane.GetCurrentPath())
                fb.recentDirs.Show()
                return nil
            }
        case 's':
            fb.cycleSortField()
            return nil
        case 'S':
            fb.reverseSort()
            return nil
        }
        return event
    }
    ```

    Key changes:
    - **Add at the TOP of handleGlobalKeys** (before any switch): RecentDirs visibility check that delegates to `fb.recentDirs.HandleKey(event)` — this ensures full key interception when popup is visible (D-08, Pitfall 2 prevention)
    - **Add `case 'r'`** in the Rune switch: check `fb.activePane == 1` AND `fb.remotePane.IsConnected()` before calling Show (D-07, POPUP-01)
    - **Before Show**: call `fb.recentDirs.SetCurrentPath(fb.remotePane.GetCurrentPath())` to provide current path for AUX-01 highlighting
    - **The `r` case falls through** (no return nil on fall-through) if conditions not met — the rune `r` passes to the pane's InputCapture. This is intentional: if local pane is focused, `r` is ignored by the pane and does nothing.
    - **Esc handling unchanged** — the existing Esc code already handles TransferModal. RecentDirs intercepts Esc via the top-level check before Esc reaches the existing handler.
