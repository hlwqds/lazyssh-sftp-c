    ## Step 1: 扩展 FileService 接口 (per D-10)

    在 `internal/core/ports/file_service.go` 的 `FileService` interface 中添加以下方法（D-10 决定提升到 FileService 而非仅 SFTPService）:

    ```go
    // Remove deletes a single file or empty directory.
    Remove(path string) error

    // RemoveAll recursively deletes a directory and all its contents.
    RemoveAll(path string) error

    // Rename renames or moves a file/directory within the same filesystem.
    Rename(oldPath, newPath string) error

    // Mkdir creates a single directory. Returns error if parent doesn't exist or directory already exists.
    Mkdir(path string) error

    // Stat returns file info for the given path.
    Stat(path string) (os.FileInfo, error)
    ```

    注意：`SFTPService` 已经有 `Stat` 和 `Remove` 方法。由于 `SFTPService` 嵌入 `FileService`，如果 `FileService` 新增了同签名方法，SFTPService 不需要重复声明（嵌入继承）。但如果 SFTPService 已有同名方法签名完全一致，Go 编译器不会报错（嵌入接口的方法被实现类型的方法满足）。确认编译通过即可。

    ## Step 3: 实现 SFTPClient 新增方法 (per RESEARCH code examples)

    在 `internal/adapters/data/sftp_client/sftp_client.go` 中添加，遵循现有 mutex 模式:

    ```go
    func (c *SFTPClient) RemoveAll(path string) error {
        c.mu.Lock()
        client := c.client
        c.mu.Unlock()
        if client == nil {
            return fmt.Errorf("not connected: call Connect first")
        }
        return client.RemoveAll(path)
    }

    func (c *SFTPClient) Rename(oldPath, newPath string) error {
        c.mu.Lock()
        client := c.client
        c.mu.Unlock()
        if client == nil {
            return fmt.Errorf("not connected: call Connect first")
        }
        return client.Rename(oldPath, newPath)
    }

    func (c *SFTPClient) Mkdir(path string) error {
        c.mu.Lock()
        client := c.client
        c.mu.Unlock()
        if client == nil {
            return fmt.Errorf("not connected: call Connect first")
        }
        return client.Mkdir(path)
    }
    ```

    注意：SFTPClient 已有 `Stat` 和 `Remove` 方法，无需重复实现。新增的是 `RemoveAll`、`Rename`、`Mkdir`。`FileService` 中的 `Stat` 和 `Remove` 签名与 SFTPService 已有方法完全一致，嵌入继承自动满足。
