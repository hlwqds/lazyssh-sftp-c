<objective>
修复 RecentDirs 永远为空的 bug：RemotePane 使用相对路径 "." 作为初始路径，导航后路径为 "./subdir" 形式，被 RecentDirs.Record() 的 "." 前缀过滤条件静默跳过。同时补全传输完成后记录远程目录路径的逻辑。

两个子问题：
1. **路径解析 bug**：RemotePane.currentPath 初始值为 "."，SFTP 的 ListDir 接受相对路径，但 NavigateInto 拼接后产生 "./foo" 形式。Record() 第59行 `strings.HasPrefix(normalized, ".")` 过滤了所有这些路径。
2. **遗漏记录点**：文件传输和目录传输完成后（F5/Enter），未调用 Record() 记录当前远程目录。

Purpose: 使 RecentDirs 功能真正可用——用户导航或传输文件后，弹窗能显示有效的绝对路径列表。
Output: 修改 4 个文件，添加 HomeDir 接口方法，修复路径初始化，补全传输记录逻辑。
</objective>

    ```go
    func (rp *RemotePane) ShowConnected() {
        rp.connected = true
        // 将 "." 初始路径解析为绝对路径，确保 Record() 不会过滤
        if rp.currentPath == "." {
            rp.currentPath = rp.sftpService.HomeDir()

        rp.SetSelectable(true, false)
        rp.Refresh()
    }
    ```
